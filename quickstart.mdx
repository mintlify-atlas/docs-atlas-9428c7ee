## Prerequisites

Before you begin, make sure you have:

- Python 3.10 or higher installed
- fluxer.py installed (`pip install fluxer.py`)
- A Fluxer bot token (see [Getting a Bot Token](#getting-a-bot-token))

## Your First Bot

Let's create a simple bot with a ping command:

<Steps>
  <Step title="Create a new Python file">
    Create a file named `bot.py`:
    
    ```python bot.py
    import fluxer

    bot = fluxer.Bot(command_prefix="!", intents=fluxer.Intents.default())

    @bot.event
    async def on_ready():
        print(f"Bot is ready! Logged in as {bot.user.username}")

    @bot.command()
    async def ping(ctx):
        await ctx.reply("Pong!")

    if __name__ == "__main__":
        TOKEN = "your_bot_token"
        bot.run(TOKEN)
    ```
  </Step>
  
  <Step title="Replace the token">
    Replace `"your_bot_token"` with your actual bot token.
    
    <Warning>
      Never commit your bot token to version control! Use environment variables in production.
    </Warning>
    
    For better security, use environment variables:
    
    ```python
    import os
    
    TOKEN = os.getenv("FLUXER_BOT_TOKEN")
    bot.run(TOKEN)
    ```
  </Step>
  
  <Step title="Run your bot">
    Execute the script:
    
    ```bash
    python bot.py
    ```
    
    You should see:
    ```
    Bot is ready! Logged in as YourBotName
    ```
  </Step>
  
  <Step title="Test the command">
    In any channel your bot has access to, type:
    ```
    !ping
    ```
    
    The bot should respond with:
    ```
    Pong!
    ```
  </Step>
</Steps>

## Getting a Bot Token

To get a bot token from Fluxer:

<Steps>
  <Step title="Visit the Fluxer Developer Portal">
    Go to [fluxer.app](https://fluxer.app) and navigate to the developer section.
  </Step>
  
  <Step title="Create a new application">
    Click "New Application" and give your bot a name.
  </Step>
  
  <Step title="Create a bot user">
    In your application settings, navigate to the "Bot" section and click "Add Bot".
  </Step>
  
  <Step title="Copy the token">
    Click "Copy" under the token section. This is your bot token.
    
    <Warning>
      Keep your token secret! Anyone with your token can control your bot.
    </Warning>
  </Step>
  
  <Step title="Invite the bot to your server">
    Use the OAuth2 URL generator to create an invite link for your bot.
  </Step>
</Steps>

## Understanding the Code

Let's break down what's happening in our bot:

### Creating the Bot

```python
bot = fluxer.Bot(command_prefix="!", intents=fluxer.Intents.default())
```

- `command_prefix="!"` - Commands start with `!` (e.g., `!ping`)
- `intents=fluxer.Intents.default()` - Subscribe to common gateway events

### Event Handlers

```python
@bot.event
async def on_ready():
    print(f"Bot is ready! Logged in as {bot.user.username}")
```

The `on_ready` event fires when the bot successfully connects to Fluxer. This is where you can:
- Log connection information
- Initialize databases
- Start background tasks

### Commands

```python
@bot.command()
async def ping(ctx):
    await ctx.reply("Pong!")
```

The `@bot.command()` decorator registers a command. The function name becomes the command name (`!ping`).

`ctx` is the context object (the message that triggered the command) with methods like:
- `ctx.reply()` - Reply to the message
- `ctx.author` - The user who sent the message
- `ctx.channel` - The channel where the message was sent
- `ctx.guild` - The guild (server) where the message was sent

### Running the Bot

```python
bot.run(TOKEN)
```

This blocking call:
1. Creates an event loop
2. Connects to the Fluxer gateway
3. Starts receiving and dispatching events
4. Handles cleanup on shutdown

## Adding More Commands

Let's add some more useful commands:

```python bot.py
import fluxer

bot = fluxer.Bot(command_prefix="!", intents=fluxer.Intents.default())

@bot.event
async def on_ready():
    print(f"Bot is ready! Logged in as {bot.user.username}")

@bot.command()
async def ping(ctx):
    await ctx.reply("Pong!")

@bot.command()
async def hello(ctx):
    await ctx.reply(f"Hello, {ctx.author.display_name}!")

@bot.command()
async def echo(ctx, *, message: str):
    """Repeat the user's message."""
    await ctx.reply(message)

@bot.command()
async def add(ctx, a: int, b: int):
    """Add two numbers."""
    result = a + b
    await ctx.reply(f"{a} + {b} = {result}")

if __name__ == "__main__":
    import os
    TOKEN = os.getenv("FLUXER_BOT_TOKEN")
    bot.run(TOKEN)
```

### Command with Arguments

```python
@bot.command()
async def echo(ctx, *, message: str):
    await ctx.reply(message)
```

The `*, message: str` syntax captures all remaining text after the command:
- `!echo Hello, world!` → message = "Hello, world!"

### Type Conversion

```python
@bot.command()
async def add(ctx, a: int, b: int):
    result = a + b
    await ctx.reply(f"{a} + {b} = {result}")
```

Arguments are automatically converted to the specified type:
- `!add 5 3` → a=5, b=3 (both integers)

## Using the Client Class

For more control, you can use the `Client` class instead:

```python
import fluxer

client = fluxer.Client(intents=fluxer.Intents.default())

@client.event
async def on_ready():
    print(f"Logged in as {client.user.username}")

@client.event
async def on_message(message):
    # Ignore messages from bots
    if message.author.bot:
        return
    
    # Respond to "ping"
    if message.content == "ping":
        await message.reply("Pong!")
    
    # Echo messages starting with "echo:"
    if message.content.startswith("echo:"):
        text = message.content[5:].strip()
        await message.reply(text)

if __name__ == "__main__":
    import os
    TOKEN = os.getenv("FLUXER_BOT_TOKEN")
    client.run(TOKEN)
```

<Note>
  The `Client` class requires manual message parsing. Use `Bot` for automatic command handling.
</Note>

## Environment Variables

For production bots, always use environment variables:

<Tabs>
  <Tab title="Linux/macOS">
    ```bash
    export FLUXER_BOT_TOKEN="your_token_here"
    python bot.py
    ```
  </Tab>
  
  <Tab title="Windows PowerShell">
    ```powershell
    $env:FLUXER_BOT_TOKEN="your_token_here"
    python bot.py
    ```
  </Tab>
  
  <Tab title=".env file">
    Create a `.env` file:
    ```
    FLUXER_BOT_TOKEN=your_token_here
    ```
    
    Load it with `python-dotenv`:
    ```python
    from dotenv import load_dotenv
    import os
    
    load_dotenv()
    TOKEN = os.getenv("FLUXER_BOT_TOKEN")
    ```
  </Tab>
</Tabs>

## Common Issues

### Bot doesn't respond to commands

- Check that your bot has permission to read and send messages
- Verify you're using the correct command prefix
- Ensure the bot is online and connected (check the `on_ready` log)

### "Login failure" error

- Verify your token is correct
- Make sure there are no extra spaces or quotes in your token
- Check that your bot hasn't been deleted or regenerated

### Events not firing

- Check that you're subscribing to the right intents
- Use `fluxer.Intents.all()` temporarily to debug
- Verify your event handler names are correct (e.g., `on_message`, not `message`)
