## Overview

The `VoiceState` class represents a user's voice state in a guild, including which channel they're connected to, mute/deaf status, and other voice-related information.

## Class Definition

```python
from fluxer.models import VoiceState

@dataclass(slots=True)
class VoiceState:
    """Represents a user's voice state in a guild."""
```

## Usage Example

```python
import fluxer

client = fluxer.Client(token="your_token")

@client.event

    """
    Called when a member's voice state changes.
    
    Args:
        member: The GuildMember whose voice state changed
        before: The VoiceState before the change
        after: The VoiceState after the change
    """
    
    # User joined a voice channel
    if before.channel_id is None and after.channel_id is not None:
        print(f"{member.display_name} joined voice channel {after.channel_id}")
        
        # Send a message to a text channel
        channel = await client.fetch_channel(NOTIFICATION_CHANNEL_ID)
        await channel.send(
            f"{member.mention} joined <#{after.channel_id}>"
        )
    
    # User left a voice channel
    elif before.channel_id is not None and after.channel_id is None:
        print(f"{member.display_name} left voice channel {before.channel_id}")
    
    # User moved between channels
    elif before.channel_id != after.channel_id:
        print(
            f"{member.display_name} moved from {before.channel_id} "
            f"to {after.channel_id}"
        )
    
    # User muted/unmuted themselves
    if before.self_mute != after.self_mute:
        if after.self_mute:
            print(f"{member.display_name} muted themselves")
```

## Voice State Properties

### Understanding Mute/Deaf States

- **`mute`**: Server muted (moderator action)
- **`deaf`**: Server deafened (moderator action)
- **`self_mute`**: User muted themselves
- **`self_deaf`**: User deafened themselves

```python
if voice_state.mute:
    print("User is server muted")
if voice_state.self_mute:
    print("User muted themselves")
```

### Streaming and Video

- **`self_stream`**: User is sharing their screen
- **`self_video`**: User has their camera on

```python
if voice_state.self_stream:
    print("User is streaming!")
if voice_state.self_video:
    print("User has video enabled")
```

### Stage Channels

For stage channels, the `suppress` and `request_to_speak_timestamp` fields are relevant:

```python
if voice_state.suppress:
    print("User is suppressed (audience member in stage)")
if voice_state.request_to_speak_timestamp:
    print(f"User requested to speak at {voice_state.request_to_speak_timestamp}")
```
