## Overview

The `User` class represents a Fluxer user account. It contains identity information and provides methods for direct messaging and user interactions.

## Class Definition

```python
from fluxer.models import User

@dataclass(slots=True)
class User:
    """Represents a Fluxer user."""
```

  ```
</ResponseField>

<ResponseField name="default_avatar_url" type="str">

  ```

<ResponseField name="banner_url" type="str | None">
  URL for the user's banner, or None if they don't have one. Automatically detects animated banners.

## Usage Example

```python
import fluxer
from fluxer import Embed

client = fluxer.Client(token="your_token")

@client.event
async def on_ready():
    print(f"Logged in as {client.user}")
    print(f"User ID: {client.user.id}")
    print(f"Created: {client.user.created_at}")

@client.event
async def on_message(message):
    # Check if author is a bot
    if message.author.bot:
        return
    
    if message.content == "!userinfo":
        user = message.author
        
        # Create an embed with user info
        embed = Embed(
            title=f"User: {user.display_name}",
            color=0x5865F2
        )
        embed.add_field(name="Username", value=user.username)
        embed.add_field(name="ID", value=str(user.id))
        embed.add_field(name="Bot", value="Yes" if user.bot else "No")
        embed.add_field(
            name="Account Created",
            value=user.created_at.strftime("%Y-%m-%d"),
            inline=False
        )
        
        # Set avatar as thumbnail
        if user.avatar_url:
            embed.set_thumbnail(url=user.avatar_url)
        
        await message.reply(embed=embed)
    
    elif message.content == "!dm":
        # Send a DM to the message author
        try:
            await message.author.send(
                "This is a direct message from the bot!"
            )
            await message.reply("Check your DMs!")
        except Exception as e:
            await message.reply(f"Couldn't send DM: {e}")

client.run()
```

## Important Notes

### Profile Fields Availability

Some fields like `bio`, `banner_hash`, and `banner_color` are ONLY available when fetching your own user via `GET /users/@me`. They are not available when fetching other users via `GET /users/{user_id}` or in guild member objects.

```python
# These fields are available
my_user = await client.fetch_user("@me")
print(my_user.bio)  # Available
print(my_user.banner_url)  # Available

# These fields are NOT available
other_user = await client.fetch_user(123456789)
print(other_user.bio)  # None (not included in response)
```
