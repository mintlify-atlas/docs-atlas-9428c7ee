## Client

Low-level client that connects to Fluxer and dispatches events. This gives you full control over the gateway lifecycle. For most bots, use the [Bot](/api/bot) subclass instead.

### Constructor

```python
Client(
    *,
    intents: Intents = Intents.default(),
    api_url: str | None = None,
    max_retries: int = 5,
    retry_forever: bool = False
)
```

<ParamField path="intents" type="Intents" default="Intents.default()">
  Gateway intents to request. Controls which events your client receives.
</ParamField>

<ParamField path="api_url" type="str | None" default="None">
  Base URL for the Fluxer API (default: `https://api.fluxer.app/v1`). Use this to connect to self-hosted Fluxer instances.
</ParamField>

<ParamField path="max_retries" type="int" default="5">
  Maximum number of retries for HTTP requests.
</ParamField>

<ParamField path="retry_forever" type="bool" default="False">
  Whether to retry HTTP requests indefinitely.
</ParamField>

### Properties

## Event Registration

### event

```python
def event(func: EventHandler) -> EventHandler
```

Decorator to register an event handler. The function name determines the event.

<ParamField path="func" type="EventHandler">
  Async function to handle the event. Must start with `on_`.
</ParamField>

**Supported events:**

- `on_ready` → READY
- `on_message` → MESSAGE_CREATE
- `on_message_edit` → MESSAGE_UPDATE
- `on_message_delete` → MESSAGE_DELETE
- `on_guild_join` → GUILD_CREATE
- `on_guild_remove` → GUILD_DELETE
- `on_member_join` → GUILD_MEMBER_ADD
- `on_member_remove` → GUILD_MEMBER_REMOVE
- `on_channel_create` → CHANNEL_CREATE
- `on_channel_update` → CHANNEL_UPDATE
- `on_channel_delete` → CHANNEL_DELETE
- `on_voice_state_update` → VOICE_STATE_UPDATE
- `on_resumed` → RESUMED
- `on_raw_reaction_add` → MESSAGE_REACTION_ADD
- `on_raw_reaction_remove` → MESSAGE_REACTION_REMOVE
- `on_raw_reaction_clear` → MESSAGE_REACTION_REMOVE_ALL
- `on_raw_reaction_clear_emoji` → MESSAGE_REACTION_REMOVE_EMOJI

**Example:**

```python
@client.event
async def on_message(message):
    print(f"Message: {message.content}")
```

### on

```python
def on(event_name: str) -> Callable[[EventHandler], EventHandler]
```

Decorator to register an event handler with an explicit name.

<ParamField path="event_name" type="str">
  The event name (without the `on_` prefix).
</ParamField>

**Example:**

```python
@client.on("message")
async def handle_msg(message):
    print(f"Message: {message.content}")
```

## Connection Lifecycle

### run

```python
def run(token: str) -> None
```

Blocking call that connects to Fluxer and runs the bot. This is the simplest way to start your bot.

<ParamField path="token" type="str" required>
  Your bot token from the Fluxer developer portal.
</ParamField>

**Example:**

```python
client = Client()

@client.event
async def on_ready():
    print(f"Logged in as {client.user.username}")

client.run("your_token_here")
```

### start

```python
async def start(token: str) -> None
```

Connect to Fluxer and start receiving events (async version). Use this if you're managing your own event loop.

<ParamField path="token" type="str" required>
  Your bot token.
</ParamField>

### close

```python
async def close() -> None
```

Disconnect from the gateway and clean up resources.
